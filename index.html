Why I Build왜 만드는지

I am motivated by concrete inefficiency, not abstract perfection. 추상적인 완성도보다, 눈앞의 비효율이 저를 움직입니다.

The pattern across my projects is consistent: I notice people spending energy on repetitive, low-value work, then I try to turn that frustration into a usable system. That is the thread connecting workflow automation, internal tooling, LLM experiments, and research. 제가 해 온 작업에는 공통된 출발점이 있습니다. 사람들이 반복적이고 의미 없는 일에 시간을 쓰는 장면을 보면, 그 비효율을 실제로 쓰이는 시스템으로 바꾸고 싶어집니다. 워크플로 자동화, 내부 툴, LLM 실험, 연구가 한 흐름으로 이어지는 이유도 여기에 있습니다.

Production Platform Migration프로덕션 플랫폼 마이그레이션

Airflow 3.0 Migration

Sep 2025 - Feb 2026 2025년 9월 - 2026년 2월

At Bagelcode, I led an Airflow 3.0 migration for a production data platform. The work included planning the upgrade path, handling operational risk in a live environment, and resolving a PostgreSQL incident that surfaced during the migration when a long-running pg_dump blocked autovacuum and drove DB CPU to 99%. I restored stability and changed the backup strategy to RDS snapshots so the migration could land on a more reliable operational setup. Bagelcode에서 프로덕션 데이터 플랫폼의 Airflow 3.0 마이그레이션을 직접 주도했습니다. 라이브 환경에서 업그레이드 경로를 정리하고 운영 리스크를 관리하는 과정이 포함됐고, 진행 중에는 장시간 실행된 pg_dump가 PostgreSQL autovacuum을 막아 DB CPU가 99%까지 치솟는 장애도 대응해야 했습니다. 문제를 진단하고 복구한 뒤 백업 방식을 RDS 스냅샷으로 전환해, 마이그레이션 결과가 더 안정적인 운영 구조로 이어지도록 마무리했습니다.

Internal Workflow Automation내부 업무 자동화

V-Manager

2022 - 2023 2022년 - 2023년

While serving in the Air Force, I felt the need for automation on my own and built this without any external reward or formal directive. V-Manager began as my first real program: an Excel VBA tool that replaced repetitive manual checks with an automated workflow. I also used UserForms so it would feel like an actual application from the user's perspective, not just a rough spreadsheet macro. That experience mattered because it was the first time I turned irritation with inefficient work into software that other people could actually use. 공군 복무 당시, 외부 보상이나 공식 지시가 있어서가 아니라 제가 스스로 자동화의 필요성을 느껴 이 도구를 만들었습니다. V-Manager는 제가 처음으로 만든 제대로 된 프로그램이었고, 반복적인 수동 확인 업무를 자동화된 흐름으로 바꾸는 Excel VBA 도구였습니다. 또 UserForm을 활용해 단순한 매크로가 아니라 사용자 입장에서 실제 프로그램처럼 느껴지도록 구성했습니다. 비효율에 대한 불편함을 실제로 다른 사람들이 쓸 수 있는 소프트웨어로 바꿔낸 첫 경험이었다는 점에서 특히 중요한 프로젝트입니다.

Self-Hosted LLM Product셀프호스팅 LLM 제품

Mirror Me

2026 - Present 2026 - 현재

Started from a real friction point: journaling is too much effort at the end of the day. The answer was a Telegram-based LLM system that turns a casual bedtime conversation into a structured daily log, lowering the barrier to consistent self-recording. Built on a repurposed tablet, connecting Telegram Bot API with Gemini for multi-turn conversation and data structuring. Spent real time working through Linux package conflicts, memory limits, and RAG instability, which reinforced that sustainable infrastructure choices matter more than flashy demos. 일기 쓰기가 귀찮다는 실제 불편에서 출발했습니다. 잠들기 전 텔레그램 대화만으로 하루를 자연스럽게 기록할 수 있도록, 기록의 진입 장벽을 낮춘 텔레그램 기반 LLM 시스템을 설계했습니다. 재활용 태블릿에서 Telegram Bot API와 Gemini를 연동해 대화 맥락을 유지하는 파이프라인을 직접 구축했고, 리눅스 패키지 충돌과 메모리 제약, RAG 불안정성을 하나씩 해결하며 화려한 데모보다 지속 가능한 인프라 선택이 중요하다는 것을 다시 확인했습니다.

Linux Termux Telegram Gemini

Personal Automation개인 자동화

SnappyCal

2025 - Present 2025 - 현재

Built an Android app that parses reservation screenshots with OCR and an LLM, then creates calendar events. It started from my own frustration with repetitive scheduling and became a product experiment in building something distributable within real cost constraints. I tested multiple open-source Hugging Face models myself, chose one by weighing the trade-off between quality and model size, and designed the app so users could bring their own Hugging Face key and run the flow on that selected model. OCR과 LLM을 활용해 예약 화면 캡처를 해석하고 캘린더 일정으로 등록해 주는 앱을 만들었습니다. 개인적인 불편에서 출발했지만, 실제 비용 제약 안에서 배포 가능한 제품을 만드는 실험이 됐습니다. 여러 Hugging Face 오픈소스 모델을 직접 테스트해 성능과 모델 크기 사이의 트레이드오프를 비교했고, 그 판단을 바탕으로 적절한 모델을 선택했습니다. 또 사용자가 자신의 Hugging Face 키를 가져오면 그 모델로 처리할 수 있도록 BYOK 구조를 설계해 제품화 가능성을 높였습니다.
